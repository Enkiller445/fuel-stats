# Бензин: статистика по Москве (авто-сбор + дашборд)

Автономная штука: **GitHub Actions** сами, 2 раза в день, собирают данные и публикуют
**одну страницу-дашборд** (GitHub Pages) — смотреть можно из интернета, в т.ч. с телефона.
Компьютер держать включённым **не нужно**.

Собираются два источника по Москве и ближнему Подмосковью:

- **Цены** — petrolplus / «АЗС-Локатор»: медиана и коридор p10–p90 по **АИ-92, АИ-95, АИ-98,
  АИ-100, ДТ**, число доступных АЗС.
- **Наличие** — gdebenz.ru (краудсорс «где есть бензин сейчас»): сколько АЗС
  **есть / нет / очередь / мало** и какие марки в наличии.

Дашборд — автономный `index.html` (без внешних библиотек), адаптивный, тема светлая/тёмная.

---

## Настройка на GitHub (один раз, ~5 минут)

1. **Создайте пустой репозиторий** на github.com (без README/лицензии).
   Рекомендуется **public** — тогда публикация страницы бесплатна (данные несекретные —
   это публичные цены на бензин). Приватный репозиторий тоже можно, но публикация
   страницы (Pages) из приватного репо требует платного GitHub Pro.

2. **Загрузите этот проект** в репозиторий. Проще всего — двойной клик по
   **`push_to_github.bat`** и вставить URL репозитория (при первом пуше откроется вход в
   GitHub в браузере). Либо вручную: `git init && git add -A && git commit -m init &&
   git remote add origin <URL> && git push -u origin main`.

3. **Включите Actions:** вкладка **Actions** → согласиться запускать workflows.

4. **Включите публикацию:** **Settings → Pages → Build and deployment → Source:
   «GitHub Actions»** (один пункт в списке; ветку/папку выбирать не нужно).

5. **Первый запуск:** вкладка **Actions → «Collect fuel stats» → Run workflow**.
   Через 1–2 минуты дашборд будет по адресу:
   **`https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/`** — этот адрес и открывайте на телефоне
   (добавьте на домашний экран).

Дальше всё само: расписание **09:00 и 21:00 МСК** (в `.github/workflows/collect.yml`,
время cron — UTC). Каждый прогон дописывает историю и пересобирает страницу.

---

## Важно: доступность из-за рубежа

GitHub Actions работает на зарубежных серверах, а сайты российские. Обычно доступ есть,
но не гарантирован (особенно gdebenz за DDoS-Guard). Схема к этому устойчива: если
источник не ответил, **дашборд не падает** — показывает прошлые данные и пометку вверху.

Проверить после первого запуска: вкладка **Actions** → последний прогон → лог. Если один
из источников стабильно недоступен из облака — есть **запасной вариант**: собирать локально
(см. ниже) и коммитить/пушить результат, либо перенести сбор на российский сервер.

---

## Запасной вариант — локальный сбор (не обязателен)

Если облако не дотянется до сайтов, можно собирать на своём ПК:

- Разово: `python -m pip install -r requirements.txt`
- Ручной запуск: двойной клик **`run_collect.bat`** (соберёт и откроет `index.html`).
- По расписанию: `powershell -File .\setup_task.ps1` (задача `FuelStatsMoscow`, 2×/день).
  Удалить: двойной клик **`remove_task.bat`**.

## Данные — «записал и удалил», репозиторий не раздувается

Скачанные выгрузки и снимки станций на диск **не сохраняются** (используются в памяти и
отбрасываются). В git попадают только два крошечных текстовых файла:

```
data/history.csv   — одна строка на прогон, все метрики (тренды). Дозаписывается.
data/status.json   — статус/свежесть последнего прогона.
```

Сам дашборд собирается в `public/index.html` и публикуется как **артефакт GitHub Pages**
(в историю git не коммитится — папка `public/` в `.gitignore`), поэтому репозиторий
остаётся маленьким при любом числе прогонов.

## Файлы проекта

| Файл | Назначение |
|------|-----------|
| `run.py` | Оркестратор: собрать оба источника → история/снимки → дашборд (это запускает Actions) |
| `collect.py` | Сбор цен (petrolplus), `collect_prices(cfg)` |
| `collect_gdebenz.py` | Сбор наличия (gdebenz), `collect_availability(cfg)` |
| `store.py` | Запись/чтение `data/history.csv` и `data/status.json` |
| `build_dashboard.py` | Сборка единой страницы `public/index.html` |
| `viz.py` | Визуальные компоненты: палитра, интерактивные графики, KPI, стили |
| `config.json` | Область, виды топлива, пороги |
| `.github/workflows/collect.yml` | Расписание и публикация (GitHub Actions) |
| `push_to_github.bat` | Помощник первичной заливки в репозиторий |
| `run_collect.bat`, `setup_task.ps1`, `remove_task.bat` | Локальный сбор (запасной вариант) |

## API источников (справочно)

- Цены: `GET https://locator.transitcard.ru/web/v1/report/points?pointTypes=8;10&x1=<lat_min>&x2=<lat_max>&y1=<lon_min>&y2=<lon_max>[&fuelAvailability=available]` → `StationList.xls`
- Наличие: `GET https://www.gdebenz.ru/api/stations?lat1=<lat_min>&lon1=<lon_min>&lat2=<lat_max>&lon2=<lon_max>` → JSON
